Summary of Chapters 6 & 7

Conditional statements are a coding must, they dictate the logic a program has to go through and its various paths and endpoints. Conditional statements return a true or false, and code specified within the conditional scope will be executed based on said statements. These statements are useful for doing anything that requires logic in your code, such as comparing values of numbers, data integrity(making sure non-negative numbers are never entered), or otherwise accounting for datatypes(making sure strings aren't ints, floats, longs, or decimals).

Loops are frequently used to sort through or otherwise find something in an array or list. Several types of loops exist (although syntax varies between languages) that can be expressed with the keyword "for" or "foreach". They are useful for finding specific spots (indexes) in an array, or if you would like a piece of code to repeat a set amount of times. You can also terminate code at specified intervals using for loops. Personally, I find foreach-loops useful for creating list of classes, and finding a specified object of said class. I.E. foreach (ClassObj obj in objects[]) { if (obj.id == idIWant){ return obj; } }

Arrays are a datatype that store various instances of another datatype. They are quite literally a specified "list" of objects and arrays have a set "space" or length/count. Array elements are accessed by indexing. These are useful is creating databases, creating slight variations of object classes to be used later, or otherwise keeping track of object instances. Lists function in a similar way, except to my understanding they are much more dynamic in size.
